## KV Cache

> [KV Cache 深度解析：从原理到工程优化的完整指南 - 知乎](https://zhuanlan.zhihu.com/p/2016843212178882587)

KV Cache 是大模型推理加速里最常见、也最实用的一项技术。KV Cache 的作用，就是把历史 token 的 **Key / Value** **缓存**下来，后面生成时直接复用。

在标准的 Transformer 架构中，每生成一个新的 token（词元），都需要基于前面所有的历史 token 来计算注意力。这意味着模型必须重新计算整个序列的 Key（键）和 Value（值）矩阵，这会导致长度为 n 的序列产生 O(n²) 的计算复杂度。而 KV Cache 的巧妙之处就在于，它会把之前已经生成过的 token 的 K 和 V 矩阵直接存起来。当模型需要生成下一个新 token 时，它只需要计算这个新 token 对应的 K 和 V，然后直接复用缓存里旧 token 的矩阵即可。这样一来，每个 token 的注意力计算复杂度就降到了 O(n)，从而极大地提升了模型的推理速度。

### 一、为什么不缓存 Query

大模型自回归解码时，每一步只需要用当前最新token生成专属的Query，这个Query只在当前时刻做一次注意力计算就没用了，下一个token会生成全新的Query，旧的完全没法复用，缓存下来只会白白占显存还没任何加速效果；而 Key 和 Value 是所有历史上下文对应的表征，一旦计算出来就固定不变，后面每生成一个新token，都要拿新Query 去跟整套历史 K、V 做注意力匹配，所以只要把 K、V 缓存住不用重复计算，就能大幅省算力、提速度，自然只缓存 KV 不缓存 Q 。
